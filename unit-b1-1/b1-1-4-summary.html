B1.1.4 – Flowcharts

📌 Learning Objective

By the end of this lesson, you will be able to:

  • Identify and explain the purpose of standard flowchart symbols
  • Draw and interpret flowcharts for a variety of programming algorithms
  • Trace the execution flow of a given flowchart, accurately following the sequence of steps
  • Construct and utilise a trace table to track changes in variable values during flowchart execution
  • Determine the output of an algorithm by tracing its flowchart

🎯 IB Command Term: Trace

Trace – "Follow and record the effects of an algorithm step by step. In practice this means working through a flowchart symbol by symbol, updating variable values in a trace table as you go, and recording the final output."

🔑 Key Terms

TermDefinition
FlowchartA visual diagram that maps out the steps in an algorithm using standardised symbols connected by arrows.
Trace TableA table used to track changes in variable values as a flowchart is executed step by step.
AlgorithmA step-by-step procedure for solving a problem.
Execution FlowThe sequence in which steps of an algorithm are performed.
IterationThe repetition of a set of instructions (looping).
SelectionA decision point in an algorithm (if-else).
SequenceSteps performed in order, one after another.

📐 Standard Flowchart Symbols

Start/End

Start / End (Oval)

Marks where the algorithm begins or terminates. Every flowchart has exactly one Start and one End.

Process

Process / Operation (Rectangle)

Represents any action performed by the algorithm: a calculation, assignment, or other operation.

?

Decision (Diamond)

Represents a condition that evaluates to true or false (yes/no). The flow branches into two paths.

I/O

Input / Output (Parallelogram)

Represents data entering the system (user input) or data leaving the system (displayed output).

→

Flowline (Arrow)

Indicates the direction of flow between symbols. Decision branches must be labelled Yes/No.

A

Connector (Circle)

Used to connect sections of a flowchart on separate pages, or to avoid crossing flow lines.

📝 How to Trace a Flowchart

StepAction
1 Identify all variables used in the flowchart. Create a trace table with one column per variable, one column per Decision condition, and one column for Output.
2 Begin at the Start symbol. Note the input value(s) given in the question and record them in the first row.
3 Follow the flowline to the next symbol. Execute what that symbol instructs: update variable values (Process), record new input (Input), or evaluate a condition (Decision).
4 At every Decision diamond, evaluate the condition using current variable values. Record Yes or No in the Decision column. Follow the correct flowline based on the result.
5 Continue step by step until you reach the End symbol. Record any output produced by Output symbols in the Output column.
6 Check your final answer by reading the Output column. If the question asks what a specific variable equals at a certain point, read that column at that row.
📝 Trace Table Conventions: Leave a cell empty if that variable was not changed in that step. Write the value of a variable after the step that changes it. If the algorithm contains a loop, each pass through the loop produces a new row. Always include the row that records the final output.

💡 Key Takeaway

"A flowchart is a visual representation of an algorithm using standardised symbols connected by arrows. Tracing a flowchart means following its execution step by step, recording how variable values change at each stage."

Flowcharts are a core examination skill in IB Computer Science. Tracing a flowchart systematically using a trace table allows you to determine the output of an algorithm for any given input, identify errors, and understand the logic of complex programs.
