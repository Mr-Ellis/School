B1.1.1 – Problem Specification

📌 Learning Objective

By the end of this lesson, you will be able to:

  • Define the five key elements of a problem specification
  • Construct a complete problem specification for a given scenario
  • Explain why problem specifications are essential for effective problem-solving

🎯 IB Command Term: Construct

Construct – "Display information in a diagrammatic or logical form. For this topic, that means producing a complete, correctly structured specification for a given problem, not just naming the elements."

🔑 Key Terms

TermDefinition
Problem SpecificationA formal document that defines exactly what a system must do. It describes the problem and requirements for a solution, but not how it will be built.
Problem StatementA clear and concise description of the problem to be solved. Describes the current situation, who is affected, and the consequences.
ConstraintsLimitations or restrictions that must be considered (e.g., time, resources, rules). Fixed conditions the solution cannot change.
Objectives / GoalsThe broad aims of the solution. Describe what the solution should achieve in general terms.
Input SpecificationsDefines the data that the system will receive, including type, format, valid range, and source.
Output SpecificationsDefines what the system must produce, including format and destination of results.
Evaluation CriteriaSpecific, measurable, SMART standards used to test whether the solution succeeds.

📊 The Five Elements

ElementQuestion It Answers
Problem StatementWhat is the problem, and why does it matter?
ConstraintsWhat are the unchangeable limits?
Objectives & GoalsWhat is the broad purpose of the solution?
Input / Output SpecificationsWhat data goes in, and what comes out?
Evaluation CriteriaHow will we know if the solution works?

💡 Key Takeaway

"A well-defined problem specification is the foundation of an effective solution."

The failure of the Mars Climate Orbiter (1999) shows what happens when a problem specification is incomplete. NASA lost a $125 million spacecraft because two engineering teams used different units and no one had formally specified what unit the data would be in.

Don't let this be you. Specify the problem before you solve it.
